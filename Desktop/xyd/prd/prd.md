# 小约定 App - MVP 产品文档

## 一、产品定位

小约定 App 旨在帮助用户记录并执行那些日常生活中非正式的“口头约定”。通过结构化、清晰的记录方式，结合端侧AI与提醒机制，提升亲密关系与团队之间的信任与执行力。

目标用户：情侣、亲子关系、自律人群、微信群体。

## 二、MVP范围说明

本版本为最简可用版本（Minimum Viable Product），主要实现：

- 基础约定创建与执行管理
- 软提醒与硬执行两种模式
- 见证人邀请机制
- 端侧 AI 辅助整理约定内容
- 多端支持（iOS / Android / 微信小程序）
- 提供通用与场景模板

> 不包含支付系统、链上智能合约、AI自动审核证据执行

## 三、核心功能

### 1. 约定创建

- 用户发起约定：填写约定名称、内容、参与方、截止时间、执行标准等
- 可选择模板（情侣约定、亲子任务、自律目标、微信群打卡等）
- AI端侧小模型辅助用户用自然语言生成清晰结构化内容（调用阿里轻量模型，如Qwen 0.6b）

### 2. 模式选择

- **软提醒模式**：定时通知提醒执行结果，不强制履行
- **硬执行模式**：到期需上传执行结果或由他人反馈验证执行情况

### 3. 见证人机制

- 每个约定可邀请1-3位见证人（好友/家人）
- 见证人可确认约定内容、到期提醒、记录是否完成

### 4. 约定管理

- 约定详情页：展示状态、倒计时、上传的执行内容、见证人反馈
- 完成后可打卡记录、展示履约历史

## 四、技术实现

### 1. 跨平台开发

使用 Flutter 实现 iOS / Android / 微信小程序统一开发与维护。

### 2. 端侧AI

- 集成阿里轻量端侧模型（Qwen 0.6b）
- 模型用于自然语言转化为结构化约定
- 无需联网即可运行，保证隐私与体验

### 3. 数据存储

- 使用 Firebase 或腾讯云 / 阿里云 进行账户、约定与通知管理
- 每个约定以 JSON 结构储存，便于模型和前端使用

## 五、发展方向（Beyond MVP）

- 接入支付功能，用于挑战金或履约担保
- 引入智能合约，增强不可篡改性与信任
- 使用AI自动审核上传执行证据
- 生成履约排行榜或关系评分，强化互动
