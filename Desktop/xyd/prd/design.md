# 小约定App - 产品设计概要

## 一、设计理念

### 核心理念
小约定App不是冰冷的任务管理工具，而是温暖的关系维护助手。我们通过**真实关系背书**、**情感化激励**和**智能化辅助**，让每一个约定都成为增进感情的契机。

### 设计原则
1. **克制设计**：功能做减法，体验做加法
2. **情感优先**：每个交互都传递温度
3. **隐私至上**：本地优先，端侧处理
4. **降低门槛**：AI让创建约定像聊天一样简单

## 二、信息架构

```
小约定App
├── 首页（今日视图）
│   ├── 待执行约定卡片
│   ├── 快速创建入口
│   └── 智能提醒
├── 创建约定
│   ├── AI对话式创建
│   ├── 模板选择
│   └── 见证人邀请
├── 约定详情
│   ├── 执行进度
│   ├── 打卡记录
│   └── 见证人反馈
├── 我的
│   ├── 关系管理（情侣/亲子/朋友）
│   ├── 个人数据
│   └── 设置
└── 通知中心
    ├── 待办提醒
    └── 互动消息
```

## 三、核心流程设计

### 3.1 约定创建流程（3步极简）

```
步骤1：描述约定
用户："我想每天陪孩子读书"
  ↓
步骤2：AI智能补全
系统：生成结构化约定
  ↓
步骤3：邀请见证（可选）
用户：选择亲友见证
  ↓
完成创建
```

### 3.2 约定执行流程

```
每日提醒 → 打卡执行 → 上传证据(可选) → 见证确认 → 完成记录
```

### 3.3 状态流转（7种状态）

```
草稿 → 待确认 → 进行中 ⟷ 暂停
                    ↓
              完成/失败/终止
```

## 四、页面设计要点

### 4.1 首页 - 今日聚焦

**设计目标**：让用户30秒内了解今日任务

**关键元素**：
- 顶部：问候语 + 今日日期
- 主体：待执行约定卡片（大卡片设计）
- 底部：快速创建悬浮按钮

**交互设计**：
- 下拉刷新同步数据
- 左滑卡片快速打卡
- 长按卡片查看详情

### 4.2 创建页 - 对话式体验

**设计目标**：像聊天一样自然

**关键特性**：
- AI对话界面（非表单）
- 实时预览生成的约定
- 一键应用推荐模板
- 语音输入支持

### 4.3 详情页 - 故事化展示

**设计目标**：记录美好历程

**内容层次**：
1. 头部：约定标题 + 倒计时
2. 进度：可视化进度条
3. 时间轴：打卡记录流
4. 互动区：见证人评论点赞

### 4.4 我的页面 - 关系为中心

**设计目标**：展示关系成长与多维身份

**模块设计**：
- **关系空间切换**：
  - 顶部场景切换器（亲子/情侣/健身/读书等）
  - 每个关系独立的视觉主题
  - 个性化背景和装饰元素
- **关系专属数据**：
  - 情侣：甜蜜指数、纪念日、爱的时光轴
  - 亲子：成长树、能力雷达图、成长相册
  - 健身：能量值、运动轨迹、PK排行
  - 读书：知识星球、阅读进度、笔记精选
- **成就系统**（关系定制）：
  - 不同关系类型独特的成就体系
  - 里程碑式激励与故事记录

## 五、交互设计规范

### 5.1 手势交互
- **下拉**：刷新数据
- **左滑**：快捷操作
- **右滑**：返回上级
- **长按**：更多选项
- **双击**：快速打卡

### 5.2 动效设计
- **过渡动画**：300ms标准时长
- **加载动画**：骨架屏优于菊花图
- **反馈动画**：点赞心形飘散效果
- **完成动画**：彩带庆祝效果

### 5.3 微交互
- 创建成功：轻微震动反馈
- 打卡完成：音效 + 动画
- 收到鼓励：暖色渐变提示

## 六、视觉设计方向

### 6.1 设计风格
- **整体风格**：温暖、简约、有呼吸感
- **图形语言**：圆角矩形为主，避免尖锐
- **插画风格**：扁平化温馨插画

### 6.2 色彩体系

```
关系型主色调：
- 情侣粉：#FF6B9D（浪漫温馨）
- 亲子蓝：#4ECDC4（温暖成长）
- 活力橙：#FFA07A（健身活力）
- 知性棕：#8B7355（读书学习）
- 薄荷绿：#95E1D3（自我提升）

中性色：
- 主文字：#2C3E50
- 辅助文字：#95A5A6
- 背景色：#F8F9FA
```

### 6.3 字体规范
- 中文：思源黑体/苹方
- 数字：DIN Pro
- 大标题：24pt
- 正文：16pt
- 辅助文字：14pt

## 七、特色功能设计

### 7.1 AI对话式创建

**交互流程**：
```
用户输入 → AI理解 → 生成建议 → 用户确认/修改 → 完成创建
```

**设计要点**：
- 对话气泡式界面
- 打字机效果展示AI回复
- 支持语音输入
- 可编辑AI生成内容

### 7.2 亲友见证机制

**设计特点**：
- 见证人头像环绕显示
- 见证签名仪式感
- 轻量化参与（仅需3次点击）
- 见证徽章激励

### 7.3 关系数据可视化

**情侣甜蜜度**：
- 爱心进度条
- 纪念日标记
- 共同回忆相册

**亲子成长值**：
- 成长树可视化
- 能力雷达图
- 成长相册
